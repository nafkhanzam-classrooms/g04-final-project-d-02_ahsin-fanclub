BUG #15

Player cannot return to menu after death.

Current behavior:

When a player's snake dies:

- player becomes spectator/dead
- gameplay scene remains active
- return button is unavailable or non-functional
- player is trapped until match ends

Problem:

Dead players should not be forced to wait for the entire match.

Possible causes:

- PLAYER_ELIMINATED event not handled
- ResultsScene transition never triggered
- Return button callback missing
- SceneManager state not updated
- Dead state blocks UI input

Required fix:

When local player dies:

- Show "You Died" overlay
- Show:
    - Return to Menu button
    - Spectate button

Verify:

1. Player dies.
2. Click Return to Menu.
3. Room cleanup executes.
4. SceneManager switches to MenuScene.
5. User can queue for another match immediately.

Generate root-cause analysis and exact code patch.
